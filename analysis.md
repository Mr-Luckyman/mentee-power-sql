# Анализ структуры базы данных mentee_db

## Урок MP-154: Введение в SQL и DDL

### 1. Общая информация

- **База данных:** `mentee_db`
- **Схема:** `public` (стандартная схема PostgreSQL)
- **Количество таблиц:** 3 (`users`, `products`, `orders`)
- **СУБД:** PostgreSQL 15 (в Docker)

### 2. Анализ таблицы `users`

#### 2.1 Структура таблицы

| Колонка      | Тип данных     | Ограничения               | Значение по умолчанию     | Описание                                              |
|--------------|----------------|---------------------------|---------------------------|-------------------------------------------------------|
| `id`         | `integer`      | `PRIMARY KEY`, `NOT NULL` | `nextval('users_id_seq')` | Уникальный идентификатор пользователя (автоинкремент) |
| `name`       | `varchar(100)` | `NOT NULL`                | -                         | Имя пользователя (макс. 100 символов)                 |
| `email`      | `varchar(100)` | `UNIQUE`, `NOT NULL`      | -                         | Email пользователя (уникальный, обязательный)         |
| `created_at` | `timestamp`    | -                         | `now()`                   | Дата и время регистрации                              |

#### 2.2 Ограничения (Constraints)

| Имя ограничения   | Тип           | Колонка | Назначение                                    |
|-------------------|---------------|---------|-----------------------------------------------|
| `users_pkey`      | `PRIMARY KEY` | `id`    | Уникальная идентификация каждого пользователя |
| `users_email_key` | `UNIQUE`      | `email` | Защита от дубликатов email                    |

#### 2.3 Индексы

| Имя индекса       | Колонка | Тип                   | Назначение                         |
|-------------------|---------|-----------------------|------------------------------------|
| `users_pkey`      | `id`    | `B-tree` (уникальный) | Быстрый поиск по ID                |
| `users_email_key` | `email` | `B-tree` (уникальный) | Быстрый поиск по email (для входа) |

### 3. Анализ таблицы `products`

#### 3.1 Структура таблицы

| Колонка       | Тип данных      | Ограничения                      | Значение по умолчанию        | Описание                                    |
|---------------|-----------------|----------------------------------|------------------------------|---------------------------------------------|
| `id`          | `integer`       | `PRIMARY KEY`, `NOT NULL`        | `nextval('products_id_seq')` | Уникальный идентификатор товара             |
| `name`        | `varchar(100)`  | `NOT NULL`                       | -                            | Название товара                             |
| `price`       | `decimal(10,2)` | `NOT NULL`, `CHECK (price >= 0)` | -                            | Цена товара (точное число, неотрицательная) |
| `category`    | `varchar(50)`   | `NOT NULL`                       | -                            | Категория товара                            |
| `description` | `text`          | -                                | -                            | Подробное описание (неограниченная длина)   |

#### 3.2 Ограничения (Constraints)

| Имя ограничения        | Тип           | Колонка | Назначение                       |
|------------------------|---------------|---------|----------------------------------|
| `products_pkey`        | `PRIMARY KEY` | `id`    | Уникальная идентификация товара  |
| `products_price_check` | `CHECK`       | `price` | Гарантирует неотрицательную цену |

#### 3.3 Индексы

| Имя индекса     | Колонка | Тип                   | Назначение                 |
|-----------------|---------|-----------------------|----------------------------|
| `products_pkey` | `id`    | `B-tree` (уникальный) | Быстрый поиск по ID товара |

### 4. Анализ таблицы `orders`

#### 4.1 Структура таблицы

| Колонка       | Тип данных      | Ограничения                                          | Значение по умолчанию      | Описание                                          |
|---------------|-----------------|------------------------------------------------------|----------------------------|---------------------------------------------------|
| `id`          | `integer`       | `PRIMARY KEY`, `NOT NULL`                            | `nextval('orders_id_seq')` | Уникальный номер заказа                           |
| `user_id`     | `integer`       | `NOT NULL`, `FOREIGN KEY → users(id)`                | -                          | ID пользователя, совершившего заказ               |
| `product_id`  | `integer`       | `NOT NULL`, `FOREIGN KEY → products(id)`             | -                          | ID заказанного товара                             |
| `quantity`    | `integer`       | `NOT NULL`, `CHECK (quantity > 0)`                   | `1`                        | Количество единиц товара                          |
| `unit_price`  | `decimal(10,2)` | `NOT NULL`                                           | -                          | Цена за единицу на момент заказа                  |
| `total_price` | `decimal(10,2)` | `GENERATED ALWAYS AS (quantity * unit_price) STORED` |                            | Итоговая сумма заказа (вычисляется автоматически) |
| `order_date`  | `timestamp`     | -                                                    | `now()`                    | Дата и время заказа                               |
| `status`      | `varchar(20)`   | -                                                    | `'PENDING'`                | Статус заказа                                     |

#### 4.2 Ограничения (Constraints)

| Имя ограничения          | Тип           | Колонка                       | Назначение                           |
|--------------------------|---------------|-------------------------------|--------------------------------------|
| `orders_pkey`            | `PRIMARY KEY` | `id`                          | Уникальная идентификация заказа      |
| `orders_user_id_fkey`    | `FOREIGN KEY` | `user_id` -> `users.id`       | Связь с таблицей пользователей       |
| `orders_product_id_fkey` | `FOREIGN KEY` | `product_id` -> `products.id` | Связь с таблицей товаров             |
| `orders_quantity_check`  | `CHECK`       | `quantity`                    | Количество должно быть положительным |

#### 4.3 Индексы

| Имя индекса              | Колонка      | Тип                   | Назначение                         |
|--------------------------|--------------|-----------------------|------------------------------------|
| `orders_pkey`            | `id`         | `B-tree` (уникальный) | Быстрый поиск по номеру заказа     |
| `orders_user_id_fkey`    | `user_id`    | `B-tree`              | Ускорение JOIN с таблицей users    |
| `orders_product_id_fkey` | `product_id` | `B-tree`              | Ускорение JOIN с таблицей products |

---

### 5. Связи между таблицами (Foreign Keys)

```sql
-- Связь orders -> users
ALTER TABLE orders 
ADD CONSTRAINT orders_user_id_fkey 
FOREIGN KEY (user_id) REFERENCES users(id);

-- Связь orders -> products
ALTER TABLE orders 
ADD CONSTRAINT orders_product_id_fkey 
FOREIGN KEY (product_id) REFERENCES products(id);
```
### 6. Выводы и наблюдения
#### Нормализация данных:
База данных следует принципам нормализации. Каждая сущность в отдельной таблице, связи через внешние ключи.

#### Целостность данных:

**PRIMARY KEY** гарантирует уникальность каждой записи

**FOREIGN KEY** обеспечивает ссылочную целостность

**CHECK** защищает от логических ошибок (отрицательная цена, нулевое количество)

**UNIQUE** предотвращает дубликаты email

#### Производительность:

Автоматические индексы на первичных и внешних ключах

Индексы ускоряют поиск и соединения таблиц

#### Автоматизация:

**SERIAL** (автоинкремент) генерирует уникальные ID

**GENERATED COLUMN** автоматически вычисляет total_price

**DEFAULT** значения упрощают вставку данных

#### Масштабируемость:
Структура позволяет легко добавлять новые сущности (например, корзину, оплату, доставку) без изменения существующих таблиц.
